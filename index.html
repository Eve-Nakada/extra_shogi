      使い方の概要

      使い方の概要

      手番側の駒または持ち駒を選ぶと、合法手だけが盤面に表示されます。ハイライトされたマスを押すと着手します。

      • 本将棋と11x11拡張検証ルール
      • JSON保存・KIF風テキスト出力
      • WebRTC手動接続による無料通信対戦
      駒の動かし方の説明

      移動定義

      駒の動きはルールセット内の moves 配列で定義します。今後駒を増やす場合も、UI側に個別の駒名や移動処理を書かず、駒定義だけを増やします。

      step
      指定方向へ1マス移動します。金・銀・玉などに使います。
      slide
      指定方向へ、駒や盤端に当たるまで直線移動します。飛・角・香などに使います。
      jump
      途中の駒を飛び越えて指定先へ移動します。桂馬や拡張駒に使います。

      方向指定

      dx は横方向、dy は盤面そのものの縦方向、fy は自分から見た前方向です。先手と後手で前方向が反転する駒は fy を使います。

      { kind: "step", dx: 0, fy: 1 }
{ kind: "slide", dx: 1, dy: 0 }
{ kind: "jump", dx: -1, fy: 2 }

      成り・持ち駒

      成駒は promoted: true の状態フラグではなく、別の駒IDとして扱います。取られたときは capturedAs を参照し、元の駒として持ち駒へ戻します。

      P  -> TO
R  -> PR
TO.capturedAs = "P"
      盤面効果の説明

      現在の扱い

      現時点の盤面は、駒の存在、合法手、選択状態、最後の指し手、王手中の玉などを描画する静的なマスです。盤面効果はまだ勝敗や合法手に影響しません。

      今後追加する場合の方針

      盤面効果を追加する場合は、駒定義とは別に board.effects またはルールセット側の effects として定義します。UIは効果表示、ロジックは合法手・勝敗への影響を担当します。

      effects: [
  { kind: "blocked", x: 4, y: 4 },
  { kind: "bonus", x: 2, y: 2, value: 1 }
]

      効果タイプ候補

      blocked
      進入不可マス。移動・駒打ちの合法手生成で除外します。
      promotionZone
      成り条件を変えるマス。派生ルール用です。
      bonus
      点数・勝敗判定に影響するマス。持将棋や派生ルール用です。
      teleport
      特殊移動マス。通常の move とは別仕様として扱います。
      操作説明

      基本操作

      1. 手番側の盤上駒を選択します。
      2. 緑の丸が移動可能マス、赤枠が駒取り可能マスです。
      3. 持ち駒は下段または上段の持駒欄から選びます。
      4. 成れる場合は画面内の成り選択パネルで選択します。

      保存・棋譜

      1. 「ローカル保存」は同じブラウザ内へ保存します。
      2. 「JSON書出」は別端末への受け渡しやバックアップ用です。
      3. 対局メタ情報を入力してから「棋譜テキスト書出」を押すとKIF風テキストを保存できます。

      通信対戦

      1. ホストが「ホスト開始（先手）」を押して出力コードを送ります。
      2. ゲストが入力コードに貼り、「ゲスト回答作成（後手）」を押して回答コードを返します。
      3. ホストが回答コードを貼り、「回答読込」を押すと接続します。
      4. 接続できない場合は「同期要求」「疎通確認」「再接続オファー作成」を使います。

      通信対戦は不正対策なしの軽量方式です。接続コードは信頼できる相手にのみ共有してください。駒追加や盤面効果追加は、駒定義・ルールセット定義を拡張し、UI側へルールを直書きしない方針です。
