# 盤上ゲーム - 将棋HTML

HTML / CSS / JavaScript ES Modules だけで動作する、将棋系の盤上ゲームです。v1.0では、公開版としてUI、README、操作説明、既知の制限を整理しています。

## 特徴

- サーバーなし、DBなし、ログインなし
- 静的ファイルだけでGitHub Pages公開可能
- 本将棋のローカル対局
- 合法手ハイライト、成り、持ち駒、駒打ち、王手放置禁止、詰み
- 千日手、連続王手の千日手、持将棋判定
- JSON保存 / 読み込み
- localStorage保存 / 読み込み
- KIF風棋譜テキスト出力
- 11x11拡張検証ルールセット
- WebRTC DataChannel による軽量通信対戦
- 観戦、手動再接続、通信ログ、同期要求、疎通確認
- 持ち時間と1手加算
- スマホ・狭幅画面向けレスポンシブ表示

## 設計方針

このプロジェクトは、将棋専用の固定実装ではなく、将棋ルールセットを持つ盤上ゲームエンジンとして構成しています。

- 盤面サイズ可変
- 駒定義データ駆動
- UIとゲームロジックの分離
- 指し手 `move` と局面 `state` の分離
- 不正対策なし
- 無料運用前提

## ローカル起動

ES Modules を使うため、`file://` 直開きではなくローカルHTTPサーバーで開いてください。

```bash
npm run serve
```

または以下でも動作します。

```bash
python3 -m http.server 8000
```

ブラウザで開きます。

```text
http://localhost:8000
```

## テスト

```bash
npm test
```

Node.js 標準の `node:test` を使います。外部 npm パッケージは不要です。

## 基本操作

1. 手番側の盤上駒をクリックします。
2. 合法手だけが盤面に表示されます。
3. 緑の丸は移動可能マス、赤枠は駒取り可能マスです。
4. 合法手のマスをクリックすると着手します。
5. 成れる場合は確認ダイアログで成る / 成らないを選びます。
6. 持ち駒は持駒欄から選び、打てるマスをクリックします。

## 保存と棋譜

### ローカル保存

同じブラウザの localStorage に保存します。

- ローカル保存
- ローカル読込

ブラウザや端末をまたぐ保存には向きません。

### JSON保存

対局データをJSONファイルとして保存します。

- JSON書出
- JSON読込

保存内容には以下が含まれます。

- 対局メタ情報
- ルールセットID
- 手番
- 終局状態
- 時計状態
- 局面ハッシュ
- 千日手診断
- 持将棋診断
- 指し手履歴

### KIF風棋譜テキスト

対局メタ情報を入力してから、以下を使います。

- 棋譜テキスト書出
- 棋譜テキストコピー

完全なKIF互換ではなく、アプリ内の指し手履歴を読みやすく出力するKIF風テキストです。

## 軽量通信対戦

通信対戦はWebRTC DataChannelを使用します。常駐サーバーは使わず、接続コードを手動で交換します。

### 通常対局

1. ホスト側が「ホスト開始（先手）」を押します。
2. 出力コードをゲストへ送ります。
3. ゲスト側が入力コードへ貼り、「ゲスト回答作成（後手）」を押します。
4. ゲスト側の出力コードをホストへ返します。
5. ホスト側が入力コードへ貼り、「回答読込」を押します。
6. 接続後、ホスト側の局面が自動同期されます。

### 観戦

観戦者は読み取り専用です。

1. ホスト側が接続コードを作成します。
2. 観戦者が入力コードへ貼ります。
3. 「観戦回答作成」を押します。
4. 観戦者の回答コードをホストが「回答読込」で読み込みます。

### 再接続

サーバーなしのため、自動再接続ではありません。

1. 切断後、「再接続オファー作成」を押します。
2. 新しいコードを相手へ渡します。
3. 相手が回答コードを作成します。
4. 回答コードを読み込みます。
5. 必要に応じて「局面再送」または「同期要求」を使います。

## 持ち時間

1. 「持ち時間を使う」を有効化します。
2. 初期持ち時間と1手加算秒を設定します。
3. 「時計設定を反映」を押します。

着手後に着手者へ1手加算が入り、相手の時計へ切り替わります。時間切れになると時間切れ負けになります。

## ルールセット

### 本将棋

9x9の通常将棋です。

### 拡張検証将棋 11x11

盤面サイズ可変と駒追加の検証用ルールセットです。

追加駒:

- 麒麟: 8方向1マス + 縦横2マスジャンプ
- 銅将: 前3方向 + 真後ろ1マス。成ると金相当
- 走兵: 前方スライド + 左右1マス。成ると金相当

## GitHub Pages公開

`.github/workflows/pages.yml` を含めています。

GitHub側で以下を設定してください。

```text
Repository Settings
  -> Pages
  -> Build and deployment
  -> Source: GitHub Actions
```

`main` ブランチへpushすると、テスト成功後にPagesへデプロイされます。

## 既知の制限

- 通信対戦に不正対策はありません。
- WebRTC接続コードは手動交換です。
- TURNサーバーは未使用のため、ネットワーク環境によっては接続できない場合があります。
- 複数観戦者の同時接続は本格対応ではありません。
- KIF風出力は完全なKIF互換ではありません。
- 11x11拡張ルールは検証用であり、既存の古将棋ルール再現ではありません。
- サーバー権威型の勝敗検証、アカウント、レーティング、チャットはありません。

## 推奨運用

- 少人数・知人同士の対局
- GitHub Pagesでの静的公開
- JSON保存による対局記録のバックアップ
- 通信対戦時は接続コードを信頼できる相手にのみ共有
