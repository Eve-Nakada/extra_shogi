# ローカル将棋

HTML / CSS / JavaScript ES Modules だけで動く、将棋系の盤上ゲームです。

v0.5 では、サーバー、DB、ログインなしの軽量通信対戦を追加しています。通信は WebRTC DataChannel を使い、接続開始だけはオファー/回答コードを手動で交換します。不正対策は行わず、双方のブラウザが同じ合法手判定で `move` を反映する前提です。

## 現在の実装範囲

- 本将棋のローカル対局
- 合法手ハイライト
- 駒取り、持ち駒、駒打ち
- 成り、強制成り
- 王手、王手放置禁止、詰み
- 投了、初期化
- 棋譜ログ表示
- 1手戻す
- 棋譜再生
- JSON書き出し / JSON読み込み
- localStorageへのローカル保存 / 読み込み
- 11x11の拡張検証ルールセット
- 追加駒のデータ定義検証
- WebRTC DataChannel による軽量通信対戦
- 手動オファー/回答コード交換
- 指し手同期、投了同期、局面再送

## ローカル起動

```bash
python3 -m http.server 8000
```

ブラウザで以下を開きます。

```text
http://localhost:8000
```

WebRTC を使う場合は、`localhost` または HTTPS で開いてください。GitHub Pages で公開した場合は HTTPS になるため、そのまま通信対戦を試せます。

## テスト

```bash
npm test
```

Node.js の組み込みテストランナーを使います。追加の npm パッケージは不要です。

## 軽量通信対戦の使い方

1. ホスト側が「ホスト開始（先手）」を押す。
2. 出力コード欄に表示されたオファーコードをコピーして、チャットなどでゲスト側へ送る。
3. ゲスト側がオファーコードを入力コード欄へ貼り、「ゲスト回答作成（後手）」を押す。
4. ゲスト側の出力コード欄に表示された回答コードをホスト側へ送る。
5. ホスト側が回答コードを入力コード欄へ貼り、「ホストで回答読込」を押す。
6. 接続済みになったら、ホスト側が現在局面を自動送信する。

通信対戦中は、局面の不一致を避けるため、ルールセット変更、初期化、1手戻す、JSON読込、ローカル読込、棋譜再生を無効化します。

## 通信方式

- 常駐サーバーなし
- DBなし
- ログインなし
- 外部 npm パッケージなし
- WebRTC DataChannel によるブラウザ間通信
- オファー/回答コードは手動コピー
- 送信対象は基本的に `move`
- 接続直後と必要時のみ `record` 全体を同期

## ルールセット

### 本将棋

通常の9x9将棋です。

### 拡張検証将棋 11x11

盤面サイズ可変と駒定義データ駆動を検証するためのルールセットです。
標準駒に加えて以下の追加駒を持ちます。

- 麒麟: 8方向1マス + 縦横2マスジャンプ
- 銅将: 前3方向 + 真後ろ1マス。成ると金相当
- 走兵: 前方スライド + 左右1マス。成ると金相当

## 保存形式

JSON書き出しでは、盤面HTMLではなく以下を保存します。

- rulesetId
- turn
- status
- history
- move
- captured
- pieceBefore / pieceAfter

通信対戦では、この `history` または `move` を共有します。

## GitHub Pages

GitHub Pages で公開する場合は、リポジトリの Settings > Pages で Source を GitHub Actions に設定してください。
`main` ブランチへ push すると、テスト成功後にデプロイされます。

## v0.6: 再接続・観戦・持ち時間

### 再接続

サーバーを使わないため、自動再接続ではなく手動再接続です。切断後に「再接続オファー作成」を押し、生成されたコードを相手に渡してください。相手は通常の回答作成と同じ手順で回答コードを返します。

### 観戦

観戦者はホストから受け取ったオファーコードを入力し、「観戦回答作成」を押します。観戦者は読み取り専用で、盤面操作・投了・時計設定はできません。

### 持ち時間

「持ち時間を使う」を有効にし、初期持ち時間と1手加算を設定してから「時計設定を反映」を押します。時計状態はJSON保存と通信同期に含まれます。
