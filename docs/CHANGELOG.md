# Changelog

## v0.9.1

- レスポンシブ表示を改善。
- 狭幅画面でツールバー、時計、通信、メタ情報パネルを1カラム化。
- 9x9盤面は画面幅に合わせて縮小し、11x11盤面は横スクロールで表示崩れを防止。
- タッチ操作向けにボタン、持ち駒、入力欄の最小サイズを調整。
- 盤面サイズCSS変数をアプリ全体にも反映し、可変盤面UIを扱いやすくした。

## v0.8.0

- 局面ハッシュを追加。盤面、持ち駒、手番、ルールセットIDを正規化して比較する。
- 同一局面4回の千日手判定を追加。
- 連続王手の千日手を通常千日手と区別して終了できるようにした。
- 持将棋判定を追加。双方の玉の入玉状態と点数を評価し、引き分けまたは勝敗を確定する。
- JSON保存形式に positionHash、repetition、impasse の診断情報を追加。
- 千日手、局面ハッシュ、持将棋点数計算のテストを追加。

## v0.5.0

### Added

- WebRTC DataChannel による軽量通信対戦を追加
- 手動オファー/回答コード交換UIを追加
- ホスト先手 / ゲスト後手の役割管理を追加
- 接続直後の局面同期を追加
- 指し手同期、投了同期、局面再送を追加
- 通信指し手の手数・手番・合法手チェックを追加
- 通信プロトコルの自動テストを追加

### Changed

- 通信中はルールセット変更、初期化、1手戻す、読込、棋譜再生を無効化
- `npm test` は TAP レポーターと `--test-force-exit` を使用

## v0.4.0

### Added

- 11x11拡張検証ルールセットを追加
- 追加駒「麒麟」「銅将」「走兵」を追加
- ルールセット切り替えUIを追加
- 拡張盤面と追加駒の自動テストを追加

## v0.3.0

### Added

- 棋譜ログ表示を追加
- 1手戻す機能を追加
- 棋譜再生機能を追加
- JSON書き出し / JSON読み込みを追加
- localStorage保存 / 読み込みを追加
- 棋譜から局面を復元するreplay機構を追加

## v0.6.0

- WebRTC手動シグナリング方式のまま、同一対局IDを使った手動再接続フローを追加。
- 観戦者ロールを追加。観戦者は局面同期・指し手・投了・時計同期を受信するが、着手できない。
- 持ち時間機能を追加。初期持ち時間と1手加算を設定できる。
- 保存/同期レコードに時計状態を含めるように変更。
- 通信プロトコルをv2へ更新し、v1の指し手メッセージも受信可能な互換処理を維持。
- 時計・再接続コード・観戦コード検証のテストを追加。

## v0.7.0

- 通信ログパネルを追加。
- 接続診断表示を追加。
- 局面同期要求メッセージを追加。
- 疎通確認用の ping / pong メッセージを追加。
- 通信同期ずれ時に自動で同期要求を送るように改善。
- WebRTC DataChannel 名を `shogi-html-v07` に更新。
- 通信プロトコルを v3 に更新し、v1/v2 受信互換を維持。
