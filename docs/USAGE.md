# 操作説明

## ローカル対局

1. ルールセットを選びます。
2. 手番側の駒を選びます。
3. ハイライトされた合法手を選びます。
4. 成れる場合は確認ダイアログで選択します。
5. 終局するとステータス欄に結果が表示されます。

## 保存

- ローカル保存: 同じブラウザ内に保存します。
- JSON書出: ファイルとして保存します。
- JSON読込: JSON保存データを復元します。
- 棋譜テキスト書出: KIF風テキストを保存します。

## 通信対戦

1. ホストがホスト開始を押します。
2. オファーコードを相手へ渡します。
3. ゲストが回答コードを作成します。
4. ホストが回答コードを読み込みます。
5. 接続後に対局します。

接続できない場合は、同期要求、疎通確認、再接続オファー作成を使います。

## 注意

通信対戦は不正対策なしです。接続コードは信頼できる相手にだけ共有してください。

## 表示設定

盤面視点、合法手ハイライト、投了前確認、初期化前確認はブラウザのlocalStorageに保存されます。同じブラウザで再度開いた場合、前回の設定が復元されます。

## 駒の動かし方

駒の動きはルールセットの `moves` 配列で定義します。`step` は1マス移動、`slide` は直線移動、`jump` は飛び越え移動です。先手・後手で前方向を反転させる動きは `fy` を使います。

## 盤面効果

現時点では盤面効果は説明のみです。実装する場合は駒定義とは分け、ルールセット側の `effects` として管理する方針です。
